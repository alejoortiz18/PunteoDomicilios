Resultados posibles

  • Éxito completo {soporte} → OK
  • Validación fallida Fila inválida
  • Error consultando datos {soporte} → Error LEER DATOS: {msg}
  • Error enviando archivo {soporte} → Error ENVIANDO ARCHIVO: {msg}

Fases del proceso

① Validación

Verificar que Soporte y Archivo no sean nulos. Si fallan → Fila inválida.

② Almacenamiento temporal

Guardar el PDF en Path.GetTempPath() con nombre original.

③ API #1 — Datos clínicos

POST con X-API-KEY. Devuelve SoporteClinicoDto con datos del paciente, convenio y medicamentos.

④ API #2 — Envío físico

POST multipart/form-data con todos los campos clínicos + PDF (Bearer token).

⑤ Limpieza (finally)

Siempre elimina el archivo temporal, independientemente del resultado.
