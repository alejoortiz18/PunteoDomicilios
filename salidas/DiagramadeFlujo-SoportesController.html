📋 Detalles de Cada Fase

FASE 1 — Entrada y Validación

  • Recibir ProcesarManualDto (lista de ItemSoporteDto)
  • Validar: Soporte ≠ nulo/vacío
  • Validar: Archivo (IFormFile) ≠ nulo
  • Si falla → registrar "Fila inválida", pasar al siguiente
  • Si pasa → guardar PDF en Path.GetTempPath()
  • El error en un ítem NO detiene el resto del lote

FASE 2 — API Datos Clínicos (SoporteDatosService)

  • Interfaz: ISoporteDatosService
  • Header: X-API-KEY: ABC123456789 (desde config)
  • Body: { "soporte": "<id>" }
  • Retorna: (SoporteClinicoDto?, string mensaje)
  • Log OK: SoporteDatosOK | Soporte={id} | Paciente={nombre}
  • Log Err: SoporteDatosError | Status={code} | Resp={body}
POST https://api-soportes.helpharma.com.co/api/DocSoporte/soportes/DatosSoportes

FASE 3 — Envío PDF (SoporteFisicoService)

  • Interfaz: ISoporteFisicoService
  • Header: Authorization: Bearer {token} (desde config)
  • Content-Type: multipart/form-data
  • 22 campos: texto clínico + medicamentos JSON + PDF
  • idUsuario = "system" (valor fijo siempre)
  • Campo "anexo": PDF con Content-Type application/pdf
  • Retorna: (bool exito, string mensaje)
  • Log OK: SoporteFisicoOK | Soporte={id}
  • Log Err: SoporteFisicoError | Status={code} | Resp={body}
POST https://intranet.helpharma.com/api/v1/soporte/fisico

FASE 4 — Limpieza (bloque finally)

  • Se ejecuta SIEMPRE, con éxito o con error
  • Verificar si el archivo temporal existe (File.Exists)
  • Si existe → File.Delete(rutaTemp)
  • Si la eliminación falla → _logger.LogWarning(...)
  • Nunca relanzar la excepción de limpieza
  • Previene acumulación de archivos residuales en disco

FASE 5 — Resultados y Respuesta

  • Recopilar todos los mensajes del ciclo de iteración
  • Formato: {idSoporte} → {estado}
  • Asignar a ViewBag.Resultados
  • Retornar View("Index") con los resultados
  • Verificar sesión activa antes de procesar (redirect a Login)

⚠️ Estados de Resultado Posibles

  • ✅ {soporte} → OK
  • ⚠️ Fila inválida
  • 🔴 {soporte} → Error LEER DATOS: {detalle}
  • 🔴 {soporte} → Error ENVIANDO ARCHIVO: {detalle}
  • 🔴 {soporte} → Error interno
